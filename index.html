About Me

Tinkerer

I've always been fascinated with the ins-and-outs of whatever I could get my hands on: How it works, What's the purpose, What could make it better... my list of questions were endless! I found importance in discovering the functionality of the pieces, because with out them, the tool wouldn't exist.
Mysteries can only be solved with a little investigation, right?

Designer

As both a performer and technical designer in Theatre, I've learned valuable lessons in efficiency, problem-solving, and quality.
"The show must go on" stood more as a rule than an acceptance of the situation. As a result, I pushed to get the job done under immense pressure because when you hear the audience applaud, you know you've done your job right.

Artist

In a free moment, you can find me with pencil and paper or a brush and canvas.
I enjoy the freedom that comes with creating and expressing something that's not as easily done with words. I enjoy drawing and painting minimalistic settings, still lifes of objects like furniture, portraits, or abstracted landscapes.

Tic Tac Toe

JavaScript, jQuery, AJAX, Curl scripts, HTML, CSS, Bootstrap, Sass

With a retro twist, this is a Front-End single page application for the all-time classic game: Tic-Tac-Toe. The application includes user authentication capabilities such as sign-up, sign-in, change password, and sign-out, as well as back-and-forth functionality when switching between 'Player X' and 'Player O'. Each game is stored and users can reference the total number of games played under the 'Game History'. There is also the added feature of playing game themed background music! With a toggle play button, the User has to freedom to play and/or pause the music at any time.

AC Town Builder

JavaScript, jQuery, AJAX, Curl Scripts Ruby, on Rails, Handlebars, HTML, CSS, Boostrap, Sass

Inspired by Nintendo's 'Animal Crossing' franchise, my AC Town Builder is a single page application with its own unique Ruby on Rails API. Users can sign-up and/or sign-in to build a town with their own unique attributes! Newly created towns are stored within the database and the user can view all or just a single town, edit town attirbutes, and delete any of that data under their account.

Beehive

JavaScript, jQuery, Node.js, Express.js, MongoDB, Mongoose, Git, Handlebars, HTML5, CSS3, Boostrap, Sass

This event listing application was built as a team (group of 4) with a focus on work flow and multi-branch version control. The goal was to build an application where users can view events, create/update/delete their own, and RSVP to any event! With the use of our mongo database and handlebars, my team was able to create a user friendly site that's easy to navigate and references event ids to modify data and RSVP!
While working as Scrum Master for our team, I set up the initial repositories and development branch for my team and I to work off of, I implemented the listings model, sign-in and sign-out features, show listing feature, as well as styling and layout for each view in the site using Bootstrap, CSS3, Sass and jQuery.

Pigment

React, Node.js, Express.js, MongoDB, Mongoose, Amazon Web Services S3, Multer, Git, JSX, HTML5, CSS3, Boostrap, Sass

This was the capstone project for my Software Engineering Immersive with Boston's General Assembly. This application, inspired by my own personal passion for the arts, is an e-commerce site where users can utilize CRUD (create/read/update/delete) actions to peruse through existing art listings and create their own. This single page application uses Amazon Web Services and a multer to handle image uploading so users can use their own images.
